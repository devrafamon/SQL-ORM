Parte do aprendizado/projeto desenvolvido para a disciplina **Engenharia de Software II (ES2)**.

O objetivo do projeto foi aplicar conceitos de **bancos de dados, SQL e ORM** no desenvolvimento de um sistema de **e-commerce**, simulando o funcionamento de uma aplicação real com persistência de dados e modelagem de entidades.

Este repositório representa parte do processo de aprendizado em **modelagem de dados, manipulação de banco de dados e integração com aplicações de software**.

---

# 📚 Sobre o Projeto

Este projeto consiste no desenvolvimento de uma base de dados para um **sistema de e-commerce**, focando principalmente na camada de **dados e persistência**.

Durante o desenvolvimento foram trabalhados conceitos como:

* modelagem de banco de dados
* escrita de consultas SQL
* relacionamento entre entidades
* utilização de **ORM (Object-Relational Mapping)** para integrar o banco de dados com a aplicação

O objetivo principal foi compreender como sistemas reais organizam e manipulam grandes volumes de dados em aplicações comerciais.

---

# 🎯 Objetivo do Projeto

O projeto foi desenvolvido com os seguintes objetivos de aprendizado:

* compreender a **estrutura de bancos de dados relacionais**
* praticar **SQL para manipulação de dados**
* aplicar **ORM para mapear objetos da aplicação para tabelas do banco**
* entender a modelagem de entidades comuns em sistemas de e-commerce

---

# 🧠 Conceitos Aplicados

Durante o desenvolvimento do projeto foram explorados conceitos importantes da área de dados.

## 🗄️ Modelagem de Banco de Dados

Estruturação de tabelas e relacionamentos entre entidades do sistema.

Exemplos de entidades comuns em e-commerce:

* clientes
* produtos
* pedidos
* itens de pedido

---

## 🧾 SQL

Utilização da linguagem SQL para:

* criação de tabelas
* inserção de dados
* consultas
* atualização de registros
* remoção de dados

---

## 🔄 ORM (Object-Relational Mapping)

Uso de ORM para permitir que a aplicação manipule dados do banco através de **objetos da linguagem de programação**, reduzindo a necessidade de escrever SQL manualmente.

O ORM realiza o mapeamento entre:

```text
Objetos da aplicação  ↔  Tabelas do banco de dados
```

---

## 🧩 Integração Aplicação ↔ Banco de Dados

O projeto demonstra como aplicações de software utilizam um banco de dados para:

* persistir informações
* recuperar dados
* manter consistência nas operações do sistema.
